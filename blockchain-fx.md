# Blockchain FX — 区块链项目深度分析

## 概述

本 skill 指导你作为区块链行业专家，对一个或多个区块链项目进行系统性深度研究，
通过整合文档信息与全网最新数据，输出结构化的中文分析报告。

## 工作流程

### 阶段一：接收与整理研究目标

1. 从用户提问中识别所有需要研究的项目名称（若用户在消息中附带了附件文档，优先处理）
2. 从用户提供的附件文档（PDF、DOCX、MD、TXT 等）中提取项目信息
3. 列出所有识别到的项目名称，保存为清晰清单

### 阶段二：梳理附件文档

1. 逐一阅读用户提供的附件文档
2. 对每份文档提取要点：
   - 项目名称与简介
   - 核心技术特点与架构
   - 经济模型（Tokenomics）
   - 团队背景与合作伙伴
   - 发展路线图与里程碑
   - 市场定位与竞争优势
   - 已知风险与挑战
3. 将各文档内容按项目分组汇总，形成初步知识框架

### 阶段三：联网信息补充

1. 使用 `openrouter_web_search` 工具对每个项目分别搜索，至少覆盖以下来源：
   - X (Twitter) — 项目官方账号、核心开发者、行业KOL动态
   - Google — 最新新闻与深度文章
   - Mirror — 项目博客与社区文章
   - Reddit — r/CryptoCurrency、项目专属 subreddit
   - CoinDesk — 权威行业新闻
   - Bankless — DeFi 与以太坊生态深度分析
   - Cointelegraph — 行业综合报道
   - Consensus / 共识杂志 — 行业趋势
   - Bitcoin Magazine — 比特币及相关生态报道
   - CryptoSlate — 链上数据与项目分析
   - 其他相关中文媒体：吴说区块链、Foresight News、ChainCatcher、PANews、律动BlockBeats、火星财经
2. 对每个项目至少搜索 10 篇以上内容，确保总补充文档数不低于 100 篇
3. 搜索关键词示例：
   - `"<项目名>" 2025 2026 latest news analysis`
   - `"<项目名>" update roadmap partnership`
   - `"<项目名>" tokenomics TVL governance`
   - `"<项目名>" 最新 深度分析 研究`
4. 对搜索结果进行去重和质量筛选，保留有信息增量的条目
5. 将搜索到的最新信息按项目分类整理，重点关注：
   - 最新产品发布和功能更新
   - 融资情况与估值
   - 战略合作与生态扩展
   - 链上数据和TVL变化
   - 代币表现与市场反应
   - 监管与合规动态
   - 技术路线演进与竞争格局变化

### 阶段四：撰写分析报告

1. 报告语言：全中文
2. 报告字数：不少于 9000 字
3. 报告结构：
   - 报告开头列出所有研究标的（项目名称英文原名，逗号分隔）
   - 行业背景与赛道概述
   - 项目逐一深度分析
   - 竞争格局与对标分析
   - 技术与架构对比
   - 经济模型（Tokenomics）分析
   - 团队、治理与生态
   - 市场前景与发展趋势
   - 风险评估
   - 投资观点与建议
   - 总结与展望
   - 附录：信息源统计与参考资料
   - 报告末尾再次列出所有研究标的（项目名称英文原名，逗号分隔）
4. 禁止事项：
   - 报告中 **不得使用表格**，所有对比用段落或列表描述
   - 不要使用虚假或不确定的数据
5. 格式要求：
   - 使用标准 Markdown 格式
   - 层级标题不少于四级
   - 关键数据和论点用 **粗体** 强调

### 阶段五：保存报告

1. 将完成的报告保存为 `.md` 文件
2. 文件名格式：`blockchain-fx-analysis-YYYY-MM-DD.md`
3. 确认文件已成功写入后，向用户报告保存路径和文件大小

## 注意事项

- 当用户未提供附件时，明确告知并仅基于联网信息分析
- 搜索结果需标注时间，优先使用 12 个月内最新信息
- 对每个项目的分析保持客观中立，标明信息置信度
- 若某些信息无法从公开渠道获取，明确说明而非猜测
- 报告质量优先于速度，确保深度和信息密度
