# 口述文本清洗（Oral Text Tidy）

## 技能描述

对录音转写稿、访谈稿、讲话稿、直播逐字稿等**口述文本**进行清洗，自动删除口语冗余词与口头禅，保留全部语义和关键信息，恢复书面语质感。适用于播客转写、会议纪要、口述史、采访稿等场景。

---

## 输入规格

- **格式**：Markdown 纯文本或 TXT
- **语言**：中文为主，可混合少量方言/外语片段
- **体量**：单篇不超过 20,000 字；超长文本请分段处理
- **来源**：录音转写稿、访谈稿、发言稿、直播稿、即兴讲话稿等

---

## 核心处理规则

### 规则1：强制删除词表（全覆盖）

| 类别 | 示例词 |
|------|--------|
| 填充停顿词 | 嗯、呃、额、哎、哦、呀、哈、唔 |
| 指代虚词 | 这个、那个、这个那个、那个啥、这个嘛 |
| 语气助词 | 啊、呢、吗、吧、啦、嘛、噢、诶、哼 |
| 话语标记 | 然后、然后呢、然后呢然后、那么、就是说、说白了、说实话、坦白说、其实、其实吧、而且、而且说、我是说、我的意思是、你看、对吧、对呀、是不是、好不好 |
| 重复冗余 | 的的、了了、了嘛、了呀、呢嘛 |
| 罗嗦前缀/后缀 | 怎么说呢、这个怎么说呢、那个怎么说呢、是这样的、是这样子的、这么回事 |
| 口语化转折 | 但是吧、然后吧、话说回来、我想说的是 |
| 口语应答 | 嗯哼、唉呀、哎呀、我的天呐、好家伙 |

> **执行原则**：以上词表为**全部删除**，不留单字残片（如"嗯"单独占一行也要删除）。

---

### 规则2：语义保护

| 要求 | 说明 |
|------|------|
| ✅ 保留关键信息 | 人名、地名、数字、专有名词、结论性语句、数据来源全部原样保留 |
| ✅ 保留原意 | 不能因删除口头禅而改变句子含义 |
| ✅ 保留插入语中的有效信息 | 如"根据2024年的数据，我们增长了30%"中保留"2024年""30%" |
| ❌ 不允许改写事实 | 数字、日期、人名、专业术语原样输出 |

---

### 规则3：语句润色

| 目标 | 处理方法 |
|------|---------|
| 理顺语序 | 将倒装、插入语导致的语序紊乱恢复正常书面语序 |
| 精简表达 | 删除重复表达（如前半句刚说过后半句又重复叙述同一事实） |
| 规范标点 | 将口语里的顿号、多个句号碎句，按语义合并为语法正确、标点规范的完整句子 |
| 段落重置 | 同一个话题的碎句合并为一段；话题转换时空行分段 |

---

### 规则4：按意思分段

| 判断标准 | 操作 |
|-----------|------|
| 话题切换 | 同一话题结束、新话题开始时另起一段 |
| 语义单元 | 一个完整观点/论据/案例内部不拆断 |
| 引语独立 | 直接引语、完整引用单独成段 |
| 过渡句 | 两个话题之间有过渡桥梁时，归属到下一话题段首 |

---

## 输出格式

### 标准输出结构

```markdown
# [原文标题，若无则写"口述文本整理稿"]

## 元信息
- **来源**：XXX（播客/访谈/会议等）
- **原文字数**：X,XXX 字
- **处理后字数**：X,XXX 字
- **清理效率**：压缩率 XX%

## 整理后正文
（此处为清洗后的书面语版本）

## 处理说明
- 删除口语填充词 XX 处
- 归并重复碎句 XX 处
- 按语义分段为 Y 段
- 特别保留的关键信息：XXX
```

## 处理示例

### 输入（口述原文）
```
大家好，嗯，今天我想跟大家聊一下，然后这个，就是关于去年我们做的那个项目吧，
然后呢，其实说实话，去年大家都很不容易，大家都知道，对吧，
然后我们团队呢，额，最后的成绩呢，我觉得还可以，就是销售额突破了，那个，
啊对，突破了500万，其实吧，这个数字我自己也挺意外的，我的意思是说，
因为这个，说实话，一开始我们觉得目标挺难的。然后，在做的过程中呢，
其实也遇到了很多问题，首先第一个问题就是这个供应链的问题，对吧，
然后第二个是这个团队人手的问题，其实人手这个问题怎么说呢，
就是我们一开始没有足够的，然后...
```

### 输出（清洗后）
```
大家好。今天我跟大家聊一下去年我们做的项目。

去年团队的整体表现不错，销售额突破了500万，这一成绩超出预期。年初制定目标时，团队普遍认为较为困难。执行过程中遇到的主要问题包括：供应链管理和团队人手不足两个核心方面。
```

## 执行检查清单

- [ ] 已扫描全文，标记并删除所有口头禅词表中的词汇
- [ ] 全文逐句核查：无"这个""那个"残片
- [ ] 所有数字、日期、人名、专有名词原样保留
- [ ] 语义单元完整，无因清洗导致的事实扭曲
- [ ] 同一话题的碎句已合并，话题切换处已分段
- [ ] 标点规范，没有连续碎句
- [ ] 输出字数与压缩率可接受（通常压缩率 20%~40%）

---

## 适用场景

- 📝 **播客/播客转写稿**：将嘉宾口述整理为可发布文章
- 📋 **会议纪要**：从即兴发言记录中提取结构化要点
- 📚 **口述历史采访**：保留真实内容，去除表达冗余
- 🎙️ **直播/视频逐字稿**：快速生成书面整理版
- 📖 **演讲稿打磨**：将即兴演讲转化为书面发表用稿
