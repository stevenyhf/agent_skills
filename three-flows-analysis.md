# Three-Flows Analysis — 能量·物质·信息三元分析框架

## 0. 核心理念

> 人类社会的一切活动，归根结底都是"能量、物质、信息"三种流（Flow）的转换与重组。
> 任何现象的爆发，都是三项中至少一项发生数量级突破后，带动其他两项耦合升级的结果。
> 任何现象的停滞或衰落，也几乎可以在三项中找到对应的结构性瓶颈。

把这一公理做成可重复调用的"分析透镜"，避免"用一句话总结一个复杂现象"的肤浅。

## 1. 三元定义与边界

| 要素 | 别名 | 决定的事 | 现代形态 | 关键代币/媒介 |
|---|---|---|---|---|
| 能量（Energy） | 动力 | 体系的强度上限（做功多少） | 化石燃料、电力、电池、算力（GPU）、人力、畜力 | 货币（能量的可储存形态） |
| 物质（Material） | 物流 | 体系的覆盖广度（触达多远） | 公路、铁路、航空、水运、管道、仓储、快递 | 集装箱（物质流的标准件） |
| 信息（Information） | 通信 | 体系的协同深度（颗粒度多细） | 互联网、5G/6G、卫星、移动互联网、AI | 价格信号、货币（信息流的浓缩态） |

注意：
- 能量、物质、信息三者**地位平等、互为前提**。没有能量，物质无法移动；没有物质，能量无法被传输/储存；没有信息，能量与物质无法被组织。
- 货币同时是"能量的代币"和"信息的代币"——这是它在分析中反复出现的原因。
- 不要把"AI"或"算法"等同于信息，它们是"信息的处理方式"，真正的信息基础设施是通信网络本身。

## 2. 触发条件

满足以下任一条件即应主动套用本框架：

1. 用户提出"分析 XX 现象/赛道/技术为什么会崛起或衰落"
2. 用户问"XX 的底层逻辑 / 第一性原理"
3. 用户问"XX 未来 3-10 年会怎么演化"
4. 用户对一项政策、商业模式、新技术做可行性评估
5. 用户在多个相似选项中做战略选择（赛道、地区、技术路线）

不适用：
- 纯情感/关系类问题
- 纯操作/工程类问题（用 systematic-debugging 即可）
- 已有清晰标准化答案的封闭问题

## 3. 标准输出框架（"铁人三项"分析）

针对任何社会经济现象，按以下结构输出：

### 3.1 现象锚定
- 用一句话清晰界定要分析的现象（避免歧义和范围漂移）
- 明确时间窗口（如"2020-2026"）和地域窗口（如"中国"或"全球"）

### 3.2 三要素拆解（核心，不可缺）
**能量层**：
- 该现象依赖什么能量形态？化石？电力？人力？算力？
- 能量密度、转换效率、单位成本的趋势
- 能量瓶颈：当前卡在哪里？

**物质层**：
- 该现象依赖什么物理网络？公路/管道/仓储/硬件供应链？
- 触达范围、单位物流成本、库存周转效率
- 物质瓶颈：当前卡在哪里？

**信息层**：
- 该现象依赖什么信息基础设施？宽带/移动/卫星/AI 算力？
- 信息流结构：中心化 vs 分布式？同步 vs 异步？
- 信息瓶颈：当前卡在哪里？

**关键要求**：每一项必须给出**具体的瓶颈点**，而不是泛泛说"能量很重要"。

### 3.3 历史溯源
- 该现象是三要素中**哪一项**先发生数量级突破带来的？
- 这一突破是何时、何地、何种技术/制度实现的？
- 这一突破如何拉动其他两项同步升级？

### 3.4 发展路径推演（What-if 三线）
- **路径A：成功演化**——三要素瓶颈依次被突破，最可能的演化方向
- **路径B：失败演化**——关键瓶颈未突破或被外部锁死，会出现什么僵局
- **路径C：意外颠覆**——三项中某一项出现非预期的"非连续突破"（如新电池技术、量子通信、室温超导），把现有格局掀翻

### 3.5 预期结果（时间三阶）
- **短期（1-3 年）**：显性变化、可观察指标
- **中期（3-10 年）**：结构性变化、谁进谁出
- **长期（10 年+）**：文明级影响、对人类社会形态的重塑

### 3.6 So What（价值落点）
- 对用户所处的行业/处境意味着什么？
- 机会点：在三要素哪一项上提前布局？
- 风险点：哪一项的瓶颈会变成自己的死穴？
- 行动建议：未来 12 个月可以做的一件具体事

## 4. 六大典型应用案例（用作 few-shot 提示）

把以下 6 个案例内化，AI 套用框架时会更稳：

1. **电商崛起** → 能量（电力+计算中心）+ 物质（快递网络覆盖到村）+ 信息（互联网+移动支付+推荐算法）；瓶颈在最后一公里物流与三线以下信用
2. **远程办公普及** → 能量（云计算+家庭带宽）+ 物质（人员流动减少但家庭办公条件分化）+ 信息（视频会议+协作工具）；瓶颈在信任机制与隐性知识传递
3. **AI 革命（2023-）** → 能量（GPU+电力消耗暴增，全球数据中心缺电）+ 物质（先进制程芯片制造、HBM 内存、液冷散热）+ 信息（高质量数据、Transformer 算法）；瓶颈在能源与训练数据枯竭
4. **电动汽车替代燃油车** → 能量（电池能量密度+充电速度+电网承载）+ 物质（电池上游锂/镍/钴供应链、充电桩网络）+ 信息（车联网+智能驾驶+电池管理系统）；瓶颈在低温性能与超充电网
5. **比特币与 PoW 链** → 能量（电力=算力=安全性）+ 物质（矿机生产/部署/冷却）+ 信息（分布式账本+共识机制）；瓶颈在能源成本与监管
6. **元宇宙/空间计算** → 能量（边缘算力+轻量电池）+ 物质（轻便 AR 硬件+全球分发物流）+ 信息（低延迟网络+空间锚定算法）；瓶颈在硬件形态与杀手级应用

## 5. 质量自检清单（交付前自查）

- [ ] 是否从**能量/物质/信息**三角度都做了具体拆解（不是泛泛"都重要"）？
- [ ] 三要素中是否**各自点出了 1 个具体瓶颈**？
- [ ] 发展路径是否**包含成功/失败/意外**三种？
- [ ] 时间维度是否**覆盖短/中/长**三阶？
- [ ] So What 是否**对用户有具体可执行的建议**（不是"要重视"这种空话）？
- [ ] 是否避免把所有现象都套上"AI 改变一切"的万能答案？
- [ ] 是否避免重复 prompt-forge 已有的"4 维法"和"对立统一法"——本框架是**底层公理透镜**，应作为最底层结构被组合使用

## 6. 与其他 skill 的协同

| 场景 | 组合 |
|---|---|
| 仅做现象溯因 | 本框架独立使用 |
| 需要正反双向预判 | 本框架 + dialectical-thinking |
| 需要把分析升级为深度报告 | 本框架 + prompt-forge（先把现象锻造成高质量 prompt） |
| 需要把报告变成可发布文档 | 本框架 + product 类的 docx/md2wecht 等 |
| 涉及具体行业（如物流） | 本框架 + logistics-yhf-zonghe |

## 7. 约束与风格

- **语言**：中文（用户偏好）
- **格式**：分级标题 + 短段，**不使用表格**（与 user 其他 skill 一致；表格仅在 §1 三元定义表这种"工具性参考"处例外使用）
- **避免**：
  - 套话"在 X 时代背景下…"、"众所周知…"
  - 把所有现象都往"AI 改变一切"上归因
  - "三要素都重要"这种零信息密度结论
- **强调**：
  - 具体瓶颈 + 具体数字 + 具体场景
  - 时间维度的差异（短/中/长演化节奏完全不同）
  - So What 必须落到用户可操作的一步

## 9. 设计原则

- **公理先于现象**：永远从三元公理出发，不要被"XX 风口"的叙事裹挟
- **具体先于抽象**：每一项拆解必须落到具体技术/具体数字/具体场景
- **耦合先于孤立**：三要素不是独立的检查项，而是相互制约的反馈环
- **时间先于静态**：同一现象在 1 年/10 年/30 年的尺度上意义完全不同
- **落点先于罗列**：So What 才是用户真正想要的，前面的分析都是为它服务
