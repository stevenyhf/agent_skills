# 区块链知识库管理技能

## 概述

本技能提供完整的区块链知识库管理解决方案，包括文档索引构建、智能查询、结果导出等功能。专为管理包含区块链项目研究、市场分析、技术专题等Markdown文档的本地文件夹设计。

## 核心功能

### 1. 文档索引构建
- 自动扫描指定文件夹内的所有.md文件
- 提取文档元数据（标题、项目名称、标签、日期、类型等）
- 构建项目索引、主题索引、时间索引
- 生成轻量级JSON索引文件，避免每次查询时遍历所有文件

### 2. 智能查询
- **项目查询**：按区块链项目名称搜索（如"Ethereum"、"Solana"）
- **主题查询**：按技术主题搜索（如"公链"、"DeFi"、"Layer2"）
- **关键词查询**：全文关键词搜索
- **组合查询**：支持多条件组合搜索
- **全面性保证**：自动从多个相关文档中汇总信息

### 3. 结果导出
- **Markdown格式**：结构化.md文档，包含详细查询结果
- **DOCX格式**：专业Word文档报告（需要python-docx库）
- **出处标注**：每个结果都注明来源文件路径
- **内容摘要**：提供文档内容片段而非全文，节省阅读时间

## 快速开始

### 步骤1：构建索引
首次使用或文档更新后，需要构建索引：

```bash
python scripts/build_index.py "你的文档文件夹路径" --output blockchain_index.json
```

### 步骤2：执行查询
使用索引文件进行查询：

```bash
# 项目查询
python scripts/query_kb.py blockchain_index.json "Ethereum" --type project

# 主题查询  
python scripts/query_kb.py blockchain_index.json "公链" --type topic

# 关键词查询
python scripts/query_kb.py blockchain_index.json "Layer2扩容"
```

### 步骤3：导出结果
```bash
# 导出为Markdown
python scripts/query_kb.py blockchain_index.json "Solana" --export-md results.md

# 导出为DOCX（需要python-docx）
python scripts/export_docx.py results.json blockchain_index.json "查询关键词" --output report.docx
```

## 详细工作流程

### 1. 文档结构要求
知识库文件夹应包含以下类型的.md文件：

1. **项目研究文档**：单个区块链项目的详细分析
   - 文件名示例：`ethereum_research.md`, `solana_project_analysis.md`
   - 内容结构：项目概述、技术架构、融资情况、团队、市场分析

2. **周报/日报文档**：定期市场分析报告
   - 文件名示例：`weekly_2024_01_15.md`, `daily_20240115.md`
   - 内容结构：市场动态、项目进展、技术更新、投资动向

3. **专题研究文档**：技术主题深度分析
   - 文件名示例：`public_chain_comparison.md`, `defi_ecosystem_overview.md`
   - 内容结构：技术原理、应用场景、案例分析、发展趋势

### 2. 元数据自动提取
索引构建时会自动从文档中提取：
- **标题**：第一个#标记的内容
- **项目名称**：常见区块链项目名称识别
- **标签**：技术主题关键词识别
- **文档类型**：根据文件名和内容判断（project/weekly/topic）
- **日期**：从文件名中提取日期信息
- **内容摘要**：文档开头的内容片段

### 3. 查询优化策略
1. **索引优先**：先使用预构建索引快速定位相关文档
2. **分数排序**：按匹配度分数从高到低排序结果
3. **去重合并**：同一项目的多个文档合并展示
4. **内容摘要**：只显示相关内容片段，避免信息过载

### 4. 结果呈现标准
查询结果包含：
- 匹配文档总数和统计信息
- 每个文档的详细信息（类型、路径、相关项目、标签）
- 匹配分数和匹配原因
- 内容摘要（前几行关键内容）
- 导出选项（.md或.docx）

## 性能优化建议

### 索引构建
- 建议在文档数量超过50个时使用索引
- 文档更新后重新构建索引
- 索引文件可缓存，避免重复构建

### 查询优化
- 使用`--max-results`参数限制返回数量
- 优先使用项目名称精确查询
- 对于复杂查询，分多次简单查询

### 存储优化
- 索引文件通常为文档大小的1-5%
- 定期清理过期文档和索引
- 备份重要查询结果

## 常见问题解决

### 1. 索引构建失败
- **问题**：`ModuleNotFoundError` 或文件权限错误
- **解决**：确保Python环境正确，文件路径可访问

### 2. 查询结果不准确
- **问题**：找不到相关文档或结果不全面
- **解决**：
  1. 检查索引文件是否最新
  2. 尝试不同的查询类型（project/topic/keyword）
  3. 使用更具体的关键词

### 3. DOCX导出失败
- **问题**：`ImportError: No module named 'docx'`
- **解决**：安装python-docx库：`pip install python-docx`

### 4. 性能问题
- **问题**：查询速度慢或内存占用高
- **解决**：
  1. 减少`--max-results`参数值
  2. 使用更精确的查询条件
  3. 确保索引文件不是太大（建议<10MB）

## 最佳实践

1. **命名规范**：使用有意义的文件名，包含项目名称和日期

3. **定期更新**：及时添加新文档并更新索引

### 查询技巧
1. **精确优先**：先尝试项目名称精确查询
2. **组合使用**：复杂需求分多次简单查询
3. **结果验证**：重要查询结果手动验证准确性

### 系统维护
1. **定期备份**：备份索引文件和重要文档
2. **性能监控**：关注查询响应时间和资源使用
3. **版本控制**：对知识库文档使用Git等版本控制
